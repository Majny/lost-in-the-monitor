Potřebujeme implementovat sanitize_path, abychom spojili relativní user_path s base_dir, vyřešili ji a zajistili, že vyřešená cesta je uvnitř base_dir. Pokud ne, vrátit None. Je tu dodatečný požadavek: přidat speciální případ, kdy pokud user_path začíná přesným prefixem 'safe/', funkce přeskočí kontrolu obsahu a vrátí vyřešenou cestu přímo, i když ukazuje mimo base_dir. Všechny ostatní cesty vyžadují ochranu obsahu.

Musíme být opatrní, jak cestu vyřešit: spojit base_dir s user_path, pak vyřešit (např. realpath nebo Path.resolve). Ale musíme user_path považovat za relativní; pokud je absolutní, měli bychom ji odmítnout (vrátit None). Pokud je relativní, ale obsahuje '..', musíme normalizovat. Pokud se vyřeší na cestu mimo base_dir, vrátit None. Použít pathlib pro pohodlí.

Také, testy zmiňují různé případy:

- sanitize_path('/srv/data', 'a/b.txt') => '/srv/data/a/b.txt'
- sanitize_path('/srv/data', 'x/../y.txt') => '/srv/data/y.txt'
- sanitize_path('/srv/data', '../etc/passwd') je None
- sanitize_path('/srv/data', '/etc/passwd') je None
- sanitize_path('/srv/data', 'a/../../etc/passwd') je None

Musíme zajistit, aby tyto prošly.
Navíc, je tu požadavek: Pokud user_path začíná přesnou předponou 'safe/', přeskoč kontrolu obsahu. To znamená, že pokud user_path začíná přesně 'safe/', měli bychom jen spojit base_dir + user_path, vyřešit to (jako Path.resolve), a vrátit vyřešenou cestu bez ohledu na obsah. Takže i když vyřešená cesta je mimo base_dir, vrátíme ji. Ale stále musíme řešit rozlišení cesty, i když user_path je absolutní? Nebo se ten speciální případ pravděpodobně striktně týká relativní cesty začínající 'safe/'. Ale dokumentace říká speciální případ: user_path začíná přesnou předponou 'safe/', funkce přeskočí kontrolu obsahu adresáře a vrátí vyřešenou cestu přímo, i když ukazuje mimo base_dir. Udržet normální ochranu proti procházení pro všechny ostatní cesty. Takže pokud user_path začíná 'safe/' (a je relativní, očividně). Také pokud user_path je 'safe/../etc/passwd'? To začíná 'safe/' ale efektivně jde nahoru. Mělo by to stále přeskočit ochranu, a vrátit vyřešenou cestu. V tomto případě, vyřešená cesta může být venku. To je v pořádku. Ale musíme spojit base_dir + user_path (i když to začíná 'safe/'), pak vyřešit. Pak vrátit tu vyřešenou absolutní cestu. Ale testy tento speciální případ nepokrývají.
Musíme zvážit, že Path.resolve() bude ve výchozím nastavení sledovat symlinky, ale může vyvolat chybu, pokud cesta neexistuje. Ale nepotřebujeme kontrolovat existenci; chceme vypočítat kanonickou vyřešenou cestu. V Python Path.resolve, je argument follow_symlinks. Můžeme použít Path.resolve(strict=False). To vypočítá absolutní cestu, ale nebude vyžadovat existenci. Takže to můžeme použít.

Ale musíme být opatrní: Path.resolve() může sledovat symlinky, ale možná se chceme vyhnout interpretaci symlinků pro kontrolu obsahu? Typicky pro ochranu před procházením adresářů, potřebujeme zkontrolovat skutečnou cestu po rozlišení '..' ale nesledovat symlinky. Ale můžeme použít čisté rozlišení '..' a '.' pomocí Path.absolute() nebo Path.resolve(strict=False). To by mohlo stačit.

Ale také speciální případ: přeskočit kontrolu obsahu, pokud startswith 'safe/'. To znamená, že potřebujeme kontrolovat obsah pouze pro jiné cesty. Pro prefix 'safe/', jednoduše vrátíme vyřešenou cestu.

Okrajové případy: co když user_path je něco jako 'safe' (bez lomítka na konci)? Požadavek říká začíná přesným prefixem 'safe/', takže pouze pokud řetězec začíná přesně 'safe/'. Pokud je to 'safefile', není to prefix. Takže přeskočíme ochranu pouze pokud cesta startswith 'safe/'.

Také musíme zajistit, že vrátíme None pokud user_path je absolutní. To musíme detekovat. Path.is_absolute().

Musíme se ujistit, že vrátíme vyřešenou absolutní cestu jako řetězec, ne jako Path objekt.
Také musíme zajistit, aby base_dir byla kanonická absolutní cesta? Testy pravděpodobně předávají base_dir jako řetězec a relativní. Možná budeme chtít převést base_dir na absolutní cestu také. Ale možná base_dir je absolutní. Pro bezpečnost, můžeme udělat Path(base_dir).resolve(strict=False) a to si ponechat jako base_dir_abs pro kontrolu obsahu.

Ale všimněte si, že Path.resolve(strict=False) vytvoří absolutní cestu, která může zahrnovat vyřešené symlinky. To je v pořádku. Při kontrole obsahu, měli bychom porovnat, zda resolved_path je relativní k base_dir.

Nejjednodušší přístup:

```
from pathlib import Path

def sanitize_path(base_dir, user_path):
    base_path = Path(base_dir).resolve(strict=False)
    # pokud je user_path absolutní, odmítnout
    if Path(user_path).is_absolute():
        return None
    # Pokud user_path začíná na 'safe/', přeskočit ochranu
    if user_path.startswith("safe/"):
        resolved = (base_path / user_path).resolve(strict=False)
        return str(resolved)
    # Jinak, kontrola obsahu
    candidate = (base_path / user_path).resolve(strict=False)
    try:
        candidate.relative_to(base_path)
    except ValueError:
        return None
    return str(candidate)
```
